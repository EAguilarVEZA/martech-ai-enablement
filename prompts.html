Prompt library

Good prompts are clear about the role, goal, context, constraints, and output format. These are intentionally simple so the team can adapt them quickly.

Presentation

Campaign update deck

Act as a senior marketing strategist. I need a 10-slide campaign update deck for [audience].

Context:
- Campaign goal: [goal]
- Audience: [audience]
- Current results: [metrics or notes]
- Issues or open questions: [notes]

Create:
1. Slide-by-slide outline
2. Strong takeaway headline for each slide
3. Suggested chart or visual
4. Speaker note for each slide
5. Questions I should answer before presenting
Use when you have notes but no story yet.
Research

Research synthesis

Act as a research analyst for a martech team. Summarize the notes below for a busy marketing leader.

Source notes:
[paste notes, links, transcript, or uploaded file summary]

Return:
- 5 key findings
- Why each finding matters
- Risks, caveats, or weak evidence
- Recommended next questions
- A short executive summary under 150 words
Use after desk research, calls, or stakeholder notes.
Document

Campaign brief

Turn these rough notes into a campaign brief.

Use this structure:
- Objective
- Target audience
- Audience problem or insight
- Core message
- Offer
- Channels
- Required assets
- Timeline
- Owners
- Risks and dependencies
- Approval checklist

Ask clarifying questions first if anything important is missing.

Notes:
[paste notes]
Use for intake, launches, and cross-functional handoffs.
Iteration

Improve the output

Improve the draft below.

Make it:
- Clearer for [audience]
- More concise
- More specific about business impact
- Easier to scan

Keep:
- The core facts
- The intended tone
- Any claims that are supported by evidence

Also list the top 3 changes you made and why.

Draft:
[paste draft]
Use after Claude or a teammate creates a first draft.
Artifact

Create a reusable asset

Create an artifact I can reuse with the team.

Artifact type:
[checklist, one-page guide, simple webpage, table, launch tracker, campaign template]

Audience:
[team or stakeholder]

Content to include:
[paste notes]

Make it:
- Easy to scan
- Practical for daily use
- Organized into clear sections
- Ready for us to edit and improve later
Use when the output should become a team reference.
Interview

Let Claude ask you questions

I want to create [campaign, brief, presentation, workflow, report].

Interview me first. Ask one question at a time.
Focus on:
- Business goal
- Audience
- Message
- Evidence
- Constraints
- Approvals
- Timeline
- Risks

When you have enough information, create the first draft in this format:
[format]
Use when you know the goal but not all details yet.
Cowork

Folder-based campaign review

Review the files in this campaign workspace.

Goal:
Create a launch readiness summary for [campaign name].

Look for:
- Missing assets
- Inconsistent messaging
- Unclear owners
- Tracking or UTM gaps
- Approval risks
- Questions we need to answer before launch

Create:
1. Executive summary
2. Readiness checklist
3. Risk table with owner and priority
4. Recommended next actions

Do not modify files unless I ask you to.
Use in Cowork when Claude can access a selected folder.
Code

Technical martech task

Explore first, then plan before making changes.

Task:
[describe the website, tracking, dashboard, CSV, or automation issue]

Please:
1. Inspect the relevant files and explain how the current setup works.
2. Identify risks or missing context.
3. Propose a plan.
4. Wait for my approval before editing.
5. After approval, implement and run the most relevant checks.
6. Summarize what changed and show evidence that it works.
Use in Claude Code for scripts, sites, analytics, or repo changes.

What should I ask Claude?

Pick the task and copy the recommended starting move. This helps new users choose a productive prompt instead of typing a vague request.

Recommended prompt move

Create a presentation outline

Give Claude the audience, goal, raw notes, and desired number of slides. Ask for slide titles, takeaway headlines, visuals, and missing questions.

Special words that power up Claude

These phrases tell Claude what kind of work to do: ask, structure, critique, verify, create, delegate, or preserve context. Teach the team to use them intentionally.

Ask first

“Ask clarifying questions first.”

Use when the request is incomplete and you want Claude to identify gaps before drafting.

Shape

“Return this as a table.”

Use for comparisons, content calendars, test plans, persona maps, and launch checklists.

Critique

“Pressure-test this recommendation.”

Use when you want risks, missing evidence, counterarguments, and stronger decision logic.

Reusable

“Turn this into a template.”

Use after a good output so the team can repeat the workflow with new campaign inputs.

Artifact

“Create an artifact.”

Use when you want a standalone document, single-page guide, simple app, visualization, or content you will keep editing.

Citations

“Cite sources and flag uncertainty.”

Use for research tasks, especially when web search or uploaded sources are part of the workflow.

Interview

“Interview me first.”

Use when the goal is fuzzy. Claude can ask better intake questions before creating the output.

Evidence

“Show your reasoning as assumptions and evidence.”

Use for recommendations. Ask Claude to separate facts, assumptions, and judgment calls.

Verify

“Give yourself a check you can run.”

Use in Code or Cowork-style work. Ask Claude how it will prove the task is done.

Delegate

“Use subagents to investigate.”

Use in Claude Code when research would clutter the main conversation or requires separate expert lenses.

Plan mode

“Explore first, then make a plan.”

Use before technical or multi-file work. Claude should understand the system before changing it.

Context reset

“Start fresh with this summary.”

Use when a thread has drifted. Give Claude the clean goal, decisions, constraints, and next step.

Best practices for the team

The quality of Claude’s work depends on the quality of the task definition, context, and feedback loop. These habits help the team get better results with fewer corrections.

Context

Give the real situation

Include audience, business goal, channel, timeline, constraints, source material, and what “good” looks like.

Format

Specify the output

Ask for a table, memo, slide outline, checklist, JSON, artifact, email, QA rubric, or executive summary.

Iterate

Treat the first answer as draft one

Ask Claude to shorten, sharpen, adapt to a persona, add missing risks, or create a second version with a different angle.

Verify

Ask for evidence

For research, ask for citations and uncertainty. For Code, ask for tests, screenshots, or command output. For Cowork, ask what files were reviewed.

Protect context

Split big work

Use one thread for research, one for drafting, and one for final polishing. In Code, use subagents for investigation.

Human review

Keep judgment with the team

Claude can accelerate the work, but the team owns accuracy, brand fit, legal review, stakeholder alignment, and final decisions.
