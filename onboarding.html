Mini onboarding agenda

Use this flow for your first session. It starts with familiar chat examples, then shows how Claude can become a more active work partner for documents, files, and technical martech tasks.

Facilitator note

Position yourself as the initial lead: you will test use cases, collect useful prompts, identify what is safe for team use, and help people choose the right Claude surface.

0-5

Context

Why access was granted, what problems we are trying to solve, and what “good use” looks like.

5-15

Chat basics

Show chat, file upload, iterative prompting, and how to ask for a specific output format.

15-30

Three starter demos

Create a presentation outline, summarize research, and draft a campaign document from raw notes.

30-40

Superpower prompts

Teach role, goal, context, constraints, output format, artifacts, citations, and critique prompts.

40-52

Cowork and Code preview

Explain when to use chat, Cowork, and Claude Code, then show a practical martech example for each.

52-60

Team next steps

Ask each person to pick one low-risk workflow to try this week and share one useful prompt back.

Go from easy to advanced

The team does not need to become AI experts on day one. Give them a ladder: use Claude for thinking first, then for reusable outputs, then for agent-style workflows when the work has many moving parts.

Easy

Ask and refine

Use chat to draft, summarize, rewrite, brainstorm, and compare options. Keep the task small and give Claude a clear format.

Repeatable

Create artifacts

Ask Claude to create a reusable guide, checklist, template, table, or simple interactive artifact that the team can keep editing.

Agent-style

Assign helper roles

Ask Claude to act as researcher, strategist, editor, QA reviewer, or analytics translator, then combine the outputs into one recommendation.

Advanced

Use Cowork or Code

Move to Cowork or Claude Code when Claude needs to work across files, inspect a folder, update assets, run checks, or coordinate subagents.

Starter use cases for marketing

Start with familiar work where Claude can help shape, summarize, rewrite, compare, and format. Avoid making the first session feel like a technical implementation workshop.

Deck structure

Turn messy notes into a story

Claude can create an executive narrative, recommended slide order, titles, speaker notes, and missing-data questions.

Audience fit

Make it right for the room

Ask for versions for leadership, campaign owners, agencies, or technical stakeholders.

Slide polish

Rewrite slide headlines

Give Claude the slide titles and ask for sharper takeaway headlines, simpler bullets, and a cleaner close.

Synthesis

Summarize sources

Paste notes or attach files, then ask for themes, contradictions, source caveats, and implications for the campaign.

Competitive view

Compare messaging

Use Claude to compare competitor positioning, offers, channels, and gaps that your team could exploit.

Research plan

Ask better questions

Claude can draft interview questions, survey angles, desk research plans, and decision criteria.

Briefs

Build campaign briefs

Transform intake notes into audience, objective, offer, message, channels, timeline, and approval sections.

QA

Create review checklists

Ask Claude to create launch QA, UTM, landing-page, email, and stakeholder-review checklists.

Reuse

Standardize templates

Turn one good example into a reusable template with placeholders and guidance for future requests.

Hypotheses

Clarify test ideas

Claude can turn “let’s test this” into a hypothesis, audience, metric, sample risk, and readout format.

Segmentation

Map audience angles

Ask for segments, pains, proof points, objections, and messaging hooks by funnel stage.

Post-mortems

Extract learnings

Give Claude performance notes and ask for what happened, likely reasons, next tests, and stakeholder summary.
