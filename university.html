Content & campaigns

1 · Turn messy notes into a campaign brief

Use case: You have scattered intake notes from a kickoff and need a structured brief your team and stakeholders can act on — fast.

Prompt
Turn these rough notes into a campaign brief.

Use this structure: objective, target audience, audience insight, core message, offer, channels, required assets, timeline, owners, risks/dependencies, approval checklist.

Ask me clarifying questions first if anything important is missing.

Notes:
[paste your notes]

Try it: Paste it into Claude with a real (non-sensitive) set of notes and let it ask you questions before drafting.

See an example result
Objective: Drive 200 webinar registrations from existing CRM contacts in 3 weeks.
Audience: Marketing-qualified contacts who opened the last 2 emails but haven't converted.
Insight: They want practical tactics, not a product pitch.
Core message: "Leave with 3 changes you can ship this week."
Channels: Email (3-touch), LinkedIn organic, website banner.
Assets: Landing page, 3 emails, 2 social graphics, reminder sequence.
Owners: Content (copy), Web (page), Email (sends), Analytics (UTMs).
Risks: Speaker confirmation, list fatigue.
Approval: Legal review of claims, UTM check, test send.
Clarifying questions it asked first: budget? hard date? primary CTA?
Research & strategy

2 · Summarize research into decisions

Use case: You have call notes, articles, or a transcript and need the "so what" for a busy leader.

Prompt
Act as a research analyst for a martech team. Summarize the notes below for a busy marketing leader.

Return:
- 5 key findings
- Why each finding matters
- Risks, caveats, or weak evidence
- Recommended next questions
- An executive summary under 150 words

Notes:
[paste notes, transcript, or links]

Try it: Paste real desk-research or call notes and see if the findings match your own read.

See an example result
Key findings: (1) Competitors lead with outcomes, we lead with features. (2) Mobile bounce is 2× desktop. (3) Webinars convert best but are under-promoted. (4) Sales wants shorter follow-ups. (5) Data gaps in attribution after click.
Why it matters: Messaging and mobile UX are the fastest levers; attribution gaps undercut budget decisions.
Caveats: Mobile data is 4 weeks only; competitor claims are self-reported.
Next questions: Which segment drives webinar conversion? What's the mobile drop-off step?
Executive summary: Shift messaging to outcomes, fix mobile, and promote webinars harder; meanwhile close the post-click attribution gap before reallocating spend.
Leadership & managers

3 · Build an executive deck outline

Use case: You have results and issues but no story yet, and a leadership update is due.

Prompt
Create a 10-slide executive update for [audience].

For each slide give: a takeaway headline, 2-3 supporting bullets, a suggested visual, and a speaker note.

Context:
- Goal: [goal]
- Current results: [metrics]
- Issues / open questions: [notes]

End with the 3 questions I should be ready to answer.

Try it: Give Claude your real numbers and adjust the headlines so each makes a point, not just a label.

See an example result
Slide 1 — "We're on pace, with one risk": Q-to-date 78% to goal; pipeline healthy; speaker risk on webinar. Visual: progress gauge.
Slide 2 — "Mobile is our biggest leak": 2× bounce vs desktop. Visual: funnel by device.
Slide 3 — "Outcome messaging beat features 1.6×": from last test. Visual: bar chart.
… (slides 4-10 follow)
Be ready to answer: What does the mobile fix cost? When do we see lift? What if the speaker drops?
Email & SFMC

4 · Plan a Marketing Cloud journey

Use case: You're scoping a nurture or onboarding journey and want owners and QA clear before building.

Prompt
Design a Marketing Cloud journey for [goal].

Return:
- Entry criteria
- Audience segments
- Message sequence with timing
- Decision splits and wait conditions
- Suppression / exit rules
- Success metrics
- A QA checklist with an owner for each item

Try it: Run it for a real journey, then sanity-check the suppression rules and owners.

See an example result
Entry: New contact with product interest = "demo".
Sequence: Day 0 welcome → Day 2 value email → Day 5 case study → decision split on engagement → Day 8 offer or re-engage.
Suppression: Exit on conversion; hold if in active sales cycle.
Metrics: Open, click-to-page, demo-booked, unsub.
QA checklist: Links + UTMs (Email), segment logic (Ops), content proof (Content), legal footer (Legal), test send (Email). Each line names an owner.
Web & landing pages

5 · Landing page copy + a publish QA

Use case: You need on-brand landing copy and a check that it's accessible and tracked before it goes live.

Prompt
Write landing page copy for [offer] aimed at [audience].

Include: 3 headline options, a subhead, 3 benefit blocks (header + 1-2 lines), 2 CTA variants, and a short FAQ.

Then give a pre-publish QA checklist covering accessibility, analytics/UTMs, and content review, with an owner for each.

Try it: Generate copy for a real page, then run the QA checklist against the staged version.

See an example result
Headlines: "Ship 3 changes this week" · "Practical AI for marketers" · "From notes to launch, faster."
Benefits: Save blank-page time · Pressure-test ideas · Reuse what works.
CTAs: "Save my seat" / "Get the playbook."
QA checklist: Alt text on images (Web), heading order + contrast (Web), UTMs + event tracking (Analytics), claims reviewed (Legal/Content), mobile render (Web). Owner listed per item.
CRM & Salesforce

6 · Turn a request into a clear user story

Use case: A stakeholder asked for "a thing" in Salesforce and you need a buildable, testable story.

Prompt
Turn this request into a clear user story for a Salesforce build.

Return: the user story (as a [role] I want [x] so that [y]), acceptance criteria, edge cases, data/fields touched, UAT steps, and the questions to confirm before building.

Request:
[paste the request]

Try it: Feed it a real (vague) request and use the questions it returns to go back to the stakeholder.

See an example result
Story: As a campaign manager, I want leads tagged by source automatically so that reporting is accurate without manual edits.
Acceptance: New lead gets a source value from UTM; blank UTMs flagged; existing leads unchanged.
Edge cases: Multiple UTMs, direct traffic, imported lists.
Fields: Lead.Source, custom UTM fields.
UAT: Create test leads per source; verify tag; verify report.
Confirm first: What's the source of truth when UTM and form disagree?
Data & reporting

7 · Explain a metric to stakeholders

Use case: A dashboard number keeps getting misread, and you need a plain-language explanation people trust.

Prompt
Explain [metric] to a non-technical stakeholder.

Return: a plain-language definition, how it's calculated, the 2 most common misreadings, what a good vs concerning value looks like, and 3 questions this metric should prompt.

Context:
[paste any definitions or data notes]

Try it: Run it for a metric your stakeholders argue about and paste the result into your dashboard notes.

See an example result
Metric: Cost per qualified lead (CPQL).
Definition: Spend divided by leads that meet the qualification bar — not all form fills.
Calculated: Channel spend ÷ qualified leads in the same window.
Misreadings: Counting all leads (inflates volume, lowers quality); ignoring lag between spend and qualification.
Good vs concerning: Stable or falling CPQL with steady quality is good; falling CPQL with rising disqualifications is a warning.
Questions: Is the qualification bar consistent? What's the lag? Which channel is the outlier?
PRM & physician relations

8 · Physician relations report readout

Use case: You have PRM activity data and need a readout that answers the questions leaders will actually ask.

Prompt
Turn this physician relations data into a stakeholder readout.

Return: what happened, why it likely happened, what to watch next, recommended next actions with owners, and 3 follow-up questions the original brief did not capture.

Data:
[paste the data or summary]

Try it: Use a real (de-identified) summary and check whether the "why" matches your field knowledge.

See an example result
What happened: Referrals up 12% in cardiology, flat in ortho.
Why: Two new cardiology liaisons ramped; ortho coverage had a vacancy.
Watch: Whether ortho recovers once the role is filled.
Next actions: Backfill ortho liaison (Ops), targeted ortho outreach (Field), monthly trend review (Analytics).
Follow-ups: Which practices drove the cardiology lift? Is capacity a constraint? What's the goal by quarter-end?

Experiments & analytics

10 · Design an A/B test

Use case: Someone says "let's test this" and you need a real plan with a metric and a readout.

Prompt
Turn "let's test this" into an experiment plan.

Return: a clear hypothesis, audience, primary metric, 2 guardrail metrics, sample-size / duration risks, the variants, and a readout template.

Idea:
[paste the test idea]

Try it: Run it on a real idea and pressure-test whether the primary metric truly measures success.

See an example result
Hypothesis: An outcome-led headline lifts click-to-page vs the feature-led control.
Audience: Engaged email segment, split 50/50.
Primary metric: Click-to-landing-page rate.
Guardrails: Unsubscribe rate, downstream demo bookings.
Risks: List size may need 2 weeks for significance.
Variants: A = feature headline, B = outcome headline.
Readout: Winner, lift %, confidence, guardrail check, recommendation.
Cross-functional

11 · Prep for a meeting

Use case: You have a meeting soon and want to walk in with an agenda, decisions, and the right questions.

Prompt
Prepare me for my meeting on [topic].

Use this context: [paste notes / prior emails].

Return: the meeting objective, a suggested agenda, key background points, the decisions we need to make, the questions I should ask, and a follow-up template I can send after.

Try it: Feed it your real context and bring the agenda + decisions list into the room.

See an example result
Objective: Align on launch date and owner for the mobile fix.
Agenda: Status (5m) → mobile data (10m) → decision on date (10m) → owners (5m).
Background: Mobile bounce 2×; fix scoped at ~1 sprint.
Decisions: Date, owner, what slips if we prioritize this.
Questions to ask: Who owns QA? What's the rollback? Dependencies?
Follow-up template: Decision, owner, date, next check-in.

Prompt
Improve the draft below.

Make it: clearer for [audience], more concise, more specific about business impact, and easier to scan.

Keep: the core facts, the intended tone, and any claims supported by evidence.

Then list the top 3 changes you made and why.

Draft:
[paste the draft]

Try it: Run it on a real draft, then read the "top 3 changes" to learn what made it better.

See an example result
Improved draft: tighter opening, one idea per sentence, a clear ask up front, and a scannable 3-bullet summary.
Top 3 changes: (1) Moved the request to the first line — leaders skim. (2) Cut hedging words to make impact concrete. (3) Replaced a dense paragraph with bullets so it scans in 10 seconds.
It kept your facts and tone, and flagged one unsupported claim to verify.
