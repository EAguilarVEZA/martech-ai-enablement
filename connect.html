Connect your tools

Connectors let Claude work inside the tools you already use — your files, email, calendar, and chat — with your permission. This is where Claude stops being a chat box and starts doing real work in your stack.

What it is

Claude, plugged into your stack

A connector links Claude to an outside tool using MCP, an open standard. Once connected, Claude can read — and sometimes act — in that tool, only within the permissions you grant.

Why it matters

Less copy-paste, more done

Pull a brief from Drive, summarize a Slack thread, draft a reply from an email, or check a calendar — without leaving Claude or shuttling content back and forth.

Two kinds

Directory vs custom

Most people use the built-in Connectors Directory (one click and sign in). Technical teams can add a custom connector by pointing Claude at a remote MCP server URL.

What you can connect

Common connectors

  • Files & docs — Google Drive, OneDrive / SharePoint
  • Email & calendar — Gmail, Outlook / Calendar
  • Chat — Slack, Microsoft Teams
  • Work tracking — Asana, Jira, Linear
  • Dev & data — GitHub and more

What's available depends on your plan and admin settings — check the directory for the current list.

How to add one

In a few clicks

  1. In Claude chat, open the Search and tools menu (the “+”), then Add connectors.
  2. Browse the Connectors Directory and pick a tool.
  3. Sign in and authorize the permissions — Claude never sees your password.
  4. Toggle the connector on for a conversation and start asking.

Try it

A first connected task

Using my connected [Drive/Email], find the latest approved [campaign brief / thread about X] and give me:
- a 5-bullet summary
- the decisions already made
- the open questions
- who owns the next step
Enable the connector first, then ask.
Guardrail

Connect responsibly

  • Only connect tools you need, and review the permissions you grant.
  • Only add custom (remote MCP) connectors you trust — they can read and change data.
  • Keep sensitive patient or confidential data out of prompts; your admin controls org-wide connectors.
