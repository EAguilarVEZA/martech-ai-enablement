Extreme Ownership

Chat, Cowork, and Code

Choose the surface based on the kind of work. Use regular Claude chat for thinking and content. Use Cowork-style file work when Claude needs to operate across a selected workspace. Use Claude Code for technical tasks that involve repositories, scripts, tests, or implementation.

Surface Best for Martech example Prompt style
Claude Chat Fast thinking, drafting, summarizing, research synthesis, content variants, and artifacts. Create a campaign brief from notes, turn research into a POV, rewrite copy for three audience segments. Give role, context, constraints, output format, and ask for clarifying questions before drafting.
Cowork Multi-step file work in a selected folder or connected workspace, especially for non-technical users. Review a folder of campaign docs, create a launch checklist, organize drafts, compare assets, and produce a final readout. Give a folder goal, allowed files, desired output, and verification steps. Ask Claude to show what it changed.
Claude Code Technical martech work: websites, scripts, tracking QA, data transforms, dashboards, repo changes, tests, and PRs. Audit landing-page tracking, create a simple HTML guide, fix a form validation issue, write a script to clean campaign CSVs. Ask Claude to explore first, make a plan, implement, run checks, and summarize evidence.
Rule of thumb

Chat answers and drafts

When the output is a recommendation, memo, outline, summary, or reusable artifact, start in chat.

Rule of thumb

Cowork handles folders

When the work is “look through these files and create/update deliverables,” use Cowork if your workspace has it enabled.

Rule of thumb

Code changes systems

When the work touches code, data scripts, QA automation, tests, or a website/app, use Claude Code.

Agents and team helpers

An “agent” is best explained as a specialized helper with a job, context, tools, and a success check. The team can start with simple role prompts in chat, then later use subagents or agent teams in Claude Code for larger work.

Beginner

Role-based helper

Tell Claude: “Act as a lifecycle marketing strategist” or “Act as a landing page QA reviewer.” This gives Claude a lens for the answer.

Intermediate

Panel of helpers

Ask Claude to answer from multiple roles: strategist, analyst, copywriter, and operations lead. Then ask it to reconcile the best recommendation.

Advanced

Subagents and teams

In Claude Code, subagents can investigate in separate context windows, while agent teams can coordinate larger projects when enabled.

Agent-style chat

Create a marketing helper team

For this task, act like a small marketing working team with four roles:
1. Strategist: clarify the business goal and audience.
2. Researcher: identify evidence, assumptions, and gaps.
3. Copy lead: improve message clarity and tone.
4. Martech ops lead: check feasibility, tracking, handoff, and launch risks.

Task:
[describe the campaign, launch, or asset]

First, give each role's assessment separately. Then combine them into one recommended plan with next steps, risks, and open questions.
Use this before a review meeting or campaign kickoff.
Claude Code

Use subagents for investigation

Use subagents to investigate this martech implementation before making changes.

Goal:
[describe the issue or improvement]

Have one subagent inspect tracking and analytics implications, one inspect frontend/user experience implications, and one inspect data or integration risks.

Return:
- Findings by subagent
- Recommended implementation plan
- Files or systems likely involved
- Verification steps

Do not edit anything until I approve the plan.
Use when a technical task needs research before action.
